# 地图模式切换交互说明

## 🎯 核心设计理念

**地图替换相机画面，成为新的背景层**

- ❌ 不是新增地图区域
- ❌ 不是分屏显示
- ✅ 地图完全替代相机，作为新背景

---

## 📊 两种模式

### 模式1：相机模式（默认）
```
┌─────────────────────────┐
│   [顶部状态栏]          │
├─────────────────────────┤
│                         │
│   📹 实时相机画面       │
│      (背景层)           │
│                         │
│   [控制UI浮动在上层]    │
└─────────────────────────┘
```

**特点**：
- 背景：实时相机画面
- 图标状态：Map图标（白色/半透明）
- 用途：第一人称视角控制

### 模式2：地图模式
```
┌─────────────────────────┐
│   [顶部状态栏]          │
├─────────────────────────┤
│                         │
│   🗺️ 实时地图显示       │
│      (背景层)           │
│                         │
│   [控制UI浮动在上层]    │
└─────────────────────────┘
```

**特点**：
- 背景：全屏实时地图
- 图标状态：Camera图标（绿色高亮）
- 用途：全局视野导航

**重要**：
- ✅ UI位置不变
- ✅ UI功能不变
- ✅ 无需重新学习

## 🎬 切换动效（200ms）

### 相机 → 地图
```typescript
exit: {
  opacity: 0,
  scale: 0.95
}

initial: {
  opacity: 0,
  scale: 1.05
}

animate: {
  opacity: 1,
  scale: 1
}
```

**视觉效果**：
1. 相机画面淡出 + 微缩小（0.95x）
2. 地图淡入 + 微缩小恢复（1.05x → 1x）
3. 时长：200ms
4. 缓动：默认ease

### 地图 → 相机
```
反向相同动画
```

## 🚫 设计禁止项

### ❌ 不允许的设计

1. **不做分屏**
   ```
   ❌ ┌──────┬──────┐
      │ 相机 │ 地图 │
      └──────┴──────┘
   ```

2. **不改变UI布局**
   ```
   ❌ 控制器位置移动
   ❌ 按钮重新排列
   ❌ 状态栏变化
   ```

3. **不新开页面**
   ```
   ❌ 全屏地图页面
   ❌ 模态对话框
   ❌ 浮层覆盖
   ```

4. **不让地图占一半**
   ```
   ❌ 上下分屏
   ❌ 左右分屏
   ❌ 画中画
   ```

### ✅ 正确的做法

```
背景层 = 相机 OR 地图 (完全替换)
控制层 = 始终不变 (浮动在上方)
```

## 🎮 用户操作流程

### 场景1：切换到地图模式
```
1. 用户点击右上角 Map 图标
2. 200ms动画：相机淡出缩小
3. 地图淡入放大到正常
4. 图标变为 Camera（绿色高亮）
5. 所有控制器仍然可用
```

### 场景2：切换回相机模式
```
1. 用户点击右上角 Camera 图标（绿色）
2. 200ms动画：地图淡出缩小
3. 相机淡入放大到正常
4. 图标恢复为 Map（半透明）
5. 所有控制器仍然可用
```

### 场景3：地图模式下控制机器人
```
1. 背景显示实时地图
2. 用户操作左摇杆 → 机器人移动
3. 地图上机器人标记实时移动
4. 路径轨迹实时绘制
5. 坐标数据实时更新
```

## 💻 技术实现

### 状态管理
```typescript
const [isMapMode, setIsMapMode] = useState(false);

// false = 相机模式
// true  = 地图模式
```

### 背景切换
```typescript
<AnimatePresence mode="wait">
  {!isMapMode ? (
    <motion.div key="camera">
      <CameraStream />
    </motion.div>
  ) : (
    <motion.div key="map">
      <MapBackground />
    </motion.div>
  )}
</AnimatePresence>
```

### 按钮逻辑
```typescript
<motion.button
  onClick={() => setIsMapMode(!isMapMode)}
  className={isMapMode 
    ? 'bg-green-600'  // 地图模式：高亮
    : 'bg-white/10'   // 相机模式：半透明
  }
>
  {isMapMode ? <Camera /> : <Map />}
</motion.button>
```

## 📊 与原MapOverlay的区别

### MapOverlay（小窗口地图）
- 位置：右上角浮动
- 大小：192x176px
- 用途：辅助参考
- 可拖拽、可全屏

### MapBackground（地图模式）
- 位置：全屏背景
- 大小：854x480px（全屏）
- 用途：主视野
- 替代相机画面

**两者可共存**：
```
相机模式 + MapOverlay小窗口 ✅
地图模式 + MapOverlay小窗口 ✅ (但通常不需要)
```
