# 05 安全交互清单（Safety Interaction Map）

## 1) 急停按钮位置和交互

- 位置：主控制页顶部右侧控制区（红色圆形按钮，手掌图标）。
- 交互：点击立即触发 `onEmergencyStop`。
- 评估：符合“急停不应二次确认”的原则。
- **建议增加**：触发后进入“急停锁控态”，禁发所有运动类命令并显示全局红色告警。

## 2) 危险操作识别

以下操作属于中高风险：
- 运动/方向控制（双摇杆）
- 速度模式切换（尤其到快速）
- 运动模式与体态切换
- 动作选择（可能影响重心）
- 自动充电、自主出桩
- SDK 开关
- 云台开启（若涉及运动联动）
- 建图导航

## 3) 二次确认策略

- **不需要确认**：急停（必须立即执行）。
- **需要确认（建议）**：
  - 自动充电 `onAutoCharge`
  - 自主出桩 `onAutoUndock`
  - SDK 开关 `onSDKToggle`
  - 建图导航 `onMapNavigation`
  - 云台开启 `onGimbalEnable`（至少首次）
- **条件确认（建议）**：
  - 速度模式切到“快速”
  - 高风险动作切换
- **已存在确认**：退出应用 `ExitConfirmModal`（合理）。

## 4) 禁止控制条件

**建议增加**统一拦截：
- 未连接 / 弱网断连
- 控制权丢失
- 急停态
- 电量过低
- 诊断 `error`（warning 限速）
- 摄像头断流 + 地图不可用同时发生（失去态势感知）

## 5) 异常场景处理

- 未连接：按钮置灰 + 顶部提示“未连接，无法发送控制命令”。
- 控制权丢失：立即锁定摇杆，提示“控制权被接管/失效”。
- 网络延迟高：显示 RTT 与“高延迟”黄条，降低命令发送频率。
- 摄像头断流：在 `CameraStream` 错误态基础上增加“切换地图视图”快捷按钮。
- 地图数据异常：显示“地图数据异常，已回退摄像头主视图”。
- 低电量：限制高速/大动作，只允许返航/充电相关操作。
- 诊断 warning/error：
  - warning：限制高风险动作。
  - error：锁定控制并引导排障。

## 6) 命令失败回滚

- 模式/开关类命令：ACK失败后回滚 UI 开关与选中态。
- 运动命令失败：摇杆自动回中并提示“发送失败”。
- 跟随启动失败：状态从 `following` 回滚到 `idle`，并显示失败原因。

## 7) 安全文案建议

- 急停触发：`已触发急停，机器人进入安全锁定状态`。
- 控制权丢失：`当前无控制权，已暂停控制指令`。
- 高延迟：`网络延迟过高，控制可能存在滞后`。
- 低电量：`电量低于15%，已限制高速与高风险动作`。
- 诊断故障：`检测到故障，请先排障后再恢复控制`。

> 结论：当前原型的安全逻辑主要停留在 UI 层展示，控制门禁与风险分级不足，建议增加。
