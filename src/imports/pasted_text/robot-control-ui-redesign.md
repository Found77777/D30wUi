在当前四足机器人控制界面（安卓横屏、黑色背景、相机为主界面）中，

重新设计底部控制区域，加入一个“竖向滑动的动作选择模块”，并在左右两侧增加核心操作按钮。

--------------------------------
一、整体布局（底部三分结构）
--------------------------------

底部区域分为三部分：

左侧：体态切换（站立/匍匐）
中间：动作选择（竖向滑动）
右侧：跟随控制（开始/停止）

整体居中对齐，保持视觉平衡，不遮挡摇杆

--------------------------------
二、中间模块（核心）：竖向滑动动作选择
--------------------------------

组件类型：
- 垂直滑动选择器（vertical carousel / picker）

位置：
- 屏幕底部中间
- 在左右按钮之间

行为：
- 上下滑动切换动作
- 自动吸附（snap）到最近选项
- 当前选中项位于中间并高亮

--------------------------------
三、动作项设计（卡片）
--------------------------------

每个动作项包含：

1. 图标（简化机器人姿态示意）
2. 文案（用户友好表达）

样式：
- 竖向排列
- 中间项最大、最清晰
- 上下项缩小并降低透明度

--------------------------------
四、动作命名（用户视角）
--------------------------------

使用用户能理解的表达：

- 标准站立
- 低姿态行走
- 稳定支撑
- 贴地模式
- 越障模式

禁止使用技术术语（如前肘后肘）

--------------------------------
五、左侧模块（体态快速切换）
--------------------------------

按钮组：

[ 站立 ]
[ 匍匐 ]

布局：
- 垂直排列
- 靠左下区域

样式：
- 圆角按钮
- 当前状态高亮（白底）

--------------------------------
六、右侧模块（跟随控制）
--------------------------------

按钮组：

[ 开始跟随 ]
[ 停止跟随 ]

布局：
- 垂直排列
- 靠右下区域

样式：
- 开始：白底
- 停止：红色强调（或描边红）

--------------------------------
七、交互逻辑
--------------------------------

- 滑动中间区域 → 切换动作
- 当前动作即时生效
- 左侧按钮 → 控制体态（独立于动作）
- 右侧按钮 → 控制跟随状态

--------------------------------
八、视觉要求
--------------------------------

- 背景：纯黑（#000000）
- UI：半透明浮层
- 中央动作模块视觉优先级最高
- 左右按钮次级
- 保持科技感、简洁、低干扰

--------------------------------
九、设计目标
--------------------------------

- 用户可单手快速切换动作
- 中间滑动区具有“控制核心”的视觉感
- 左右按钮作为快速操作补充
- 整体布局稳定、对称、易理解
