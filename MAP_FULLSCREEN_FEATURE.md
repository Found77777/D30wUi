# 实时地图全屏功能说明

## ✨ 功能概述

实时地图现在支持全屏模式，点击放大按钮后会**完全替代相机画面**，成为整个应用的背景。

---

## 🎯 功能特性

### 小窗口模式（默认）
- 📍 位置：右上角小窗口（192x176px）
- 🖱️ 拖拽：可以自由拖动位置
- 🎨 背景：半透明黑色背景（bg-black/60）
- 📊 显示：基础地图信息

### 全屏模式（点击放大后）
- 📺 覆盖：完全覆盖整个屏幕，包括相机画面
- 🎨 背景：深色背景（bg-black/95），完全替代相机
- 📏 层级：z-50，位于所有元素之上（除顶栏和控制器外）
- 🔍 细节：增强的视觉元素

---

## 🎨 全屏模式独有元素

### 1. 坐标显示（左上角）
```
X: 0.00m
Y: 0.00m
朝向: 0.0°
```
- 实时显示机器人位置坐标
- 显示机器人朝向角度
- 绿色文字，等宽字体

### 2. 距离标尺（右上角）
```
|————————| 5.0m
```
- 动态缩放的距离标尺
- 根据缩放级别自动调整
- 帮助判断实际距离

### 3. 指南针（右下角）
```
   N ▲
  ◯ ○ ◯
   ▼
```
- 红色N标记指向北方
- 圆形罗盘设计
- 方便方向识别

### 4. 增强的视觉效果
- ✅ 更大的机器人标记（12x12px → 24x24px）
- ✅ 更粗的路径线（2px → 3px）
- ✅ 更亮的网格背景（opacity: 20% → 30%）
- ✅ 更大的UI按钮和文字

---

## 🎮 操作说明

### 进入全屏模式
1. 点击地图右上角的 **放大图标** (⛶)
2. 地图会平滑过渡到全屏
3. 相机画面被完全替代

### 退出全屏模式
1. 点击地图右上角的 **缩小图标** (⛶)
2. 地图恢复到小窗口
3. 相机画面重新显示

### 缩放控制
- **放大**: 点击 `+` 按钮（最大 2.0x）
- **缩小**: 点击 `-` 按钮（最小 0.5x）
- 缩放会影响网格大小和距离标尺

## 🔌 接口集成

### 传入地图数据

```typescript
<App
  mapData={{
    robotPosition: { x: 10.5, y: 20.3 },  // 机器人位置
    robotHeading: 45,                      // 朝向角度（度）
    pathPoints: [                          // 路径点
      { x: 0, y: 0 },
      { x: 10, y: 20 },
    ],
    obstacles: [                           // 障碍物
      { x: 5, y: 5, radius: 2 },
    ],
  }}
/>
```

### 地图更新回调

```typescript
<App
  onMapUpdate={(data) => {
    console.log('机器人位置:', data.position);
    console.log('机器人朝向:', data.heading);
  }}
/>
```

## 🎬 视觉层级（Z-Index）

从底到顶：
1. **相机画面** - z-0（背景层）
2. **控制器UI** - z-10~z-30（中间层）
3. **小窗口地图** - z-30（浮动层）
4. **全屏地图** - z-50（顶层）
5. **顶部状态栏** - z-auto（始终可见）

### 全屏地图时的层级
```
顶部栏（可见）
  ↓
全屏地图（z-50）
  ↓
控制器UI（被覆盖但可操作）
  ↓
相机画面（完全被遮挡）
```

## 🔧 技术实现

### CSS类名切换
```typescript
className={`${
  isFullscreen
    ? 'fixed inset-0 z-50 rounded-none border-0 bg-black/95'
    : 'absolute z-30 w-48 h-44 cursor-move bg-black/60'
}`}
```

### 条件渲染
```typescript
{isFullscreen && (
  <>
    <CoordinateDisplay />
    <DistanceScale />
    <Compass />
  </>
)}
```

### 响应式尺寸
```typescript
className={`${isFullscreen ? 'w-12 h-12' : 'w-6 h-6'}`}
```

## 📊 性能优化

- ✅ 使用CSS `position: fixed` 避免重绘
- ✅ 动画使用 `motion` 库硬件加速
- ✅ 条件渲染减少DOM节点
- ✅ 背景网格使用CSS渐变，非图片
